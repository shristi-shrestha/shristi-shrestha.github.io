My day job runs on R and command-line genomics tools, so it might seem odd that I spend some of my spare time on general software development and data visualization projects that have nothing to do with islets or RNA-seq. But the two feed each other more than I expected.

## Visualization as a second language

Every analysis eventually needs to become a figure someone outside the lab can understand. Learning to build clean, custom visualizations - rather than reaching for whatever the default plotting library gives you - has made me a better analyst, because it forces me to decide what actually matters in the data before I decide how to draw it.

## Software development as a habit

Picking up general software development practices - version control discipline, testing, thinking about how a script will be reused rather than run once - has quietly improved how I write analysis code too. A one-off script and a small internal tool aren't as different as they look; both benefit from being written for the person who has to read them later.

This site itself is part of that hobby: a small, ongoing project in data science and creative data visualization that I keep coming back to alongside the research work. You can find a few examples on the [Projects](/projects) page.
